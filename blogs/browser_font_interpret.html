How does the browser interpret width?

We said 16px is default browser width so what does it mean when we say 200px?

By default average browser window size = 1366 x 784. The width property in CSS specifies the width of the element's content area. The content area is the portion inside the padding border, and margin of an element(the box model).

By default the browser's size is 16 px which means if you did't overriden, the content area will inherit the browser's size as its size. (Done by browser) If we declare the width, then the browser's default size will be overridden and get the declared size on the browser screen.

The width calculation is : the render's width is calculated using the container block's width, the renderer's style "width" property, the margins and borders. The calculation is done by WebeKit as the following:
For example let the width be 200px.

  • The container width is the maximum of the containers available width and 0. If it is not available it will get its default size. (if it is text, its font will get the browser default size). In this case the content width is clientWidth() - paddingleft() - paddingright().
  • The elements width is the width style attribute. It will be calculated as an absloute or it will be calculated the exact value of the container width.
  • The horizontal borders and paddings are now added.
  • If you use scalable rem, em and % by using our browser and DOM settings. If you set the width of the body to 200px, then 1 rem = 16px (if the default browser size is 16px), 1 em = 200px and 100 % = 200px.
